Sanity Testing your App's Build

Your 'build' process creates both a 'minified' and a 'readable' version of your app. They should be automatically generated using the 'build task'. The 'minified' version will be used to deploy and the 'readable' version is used for code audit.

Please Sanity test both versions of your app regularly to make sure they function the same as your development version. Use the links below to Sanity test.
